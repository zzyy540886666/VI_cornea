# 代码提交与更新指南

本文档记录项目代码更改后的完整提交流程，确保 GitHub 和 Streamlit Cloud 同步更新。

---

## 一、快速提交流程（3 步）

### 步骤 1：查看当前更改

```powershell
# 在项目根目录执行
git status
```

输出示例：
```
Changes not staged for commit:
  modified:   app.py
  modified:   model_service.py
```

### 步骤 2：暂存并提交

```powershell
# 暂存所有更改
git add .

# 或只暂存特定文件
git add app.py model_service.py

# 提交（请使用规范的提交信息）
git commit -m "类型: 简短描述"
```

### 步骤 3：推送到 GitHub

```powershell
git push origin main
```

推送成功后，Streamlit Cloud 会自动检测到更新并重新部署（约 2-3 分钟）。

---

## 二、提交信息规范

采用 [Conventional Commits](https://www.conventionalcommits.org/) 格式：

```
<类型>: <描述>

[可选的详细说明]
```

### 常用类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 添加批量导出功能` |
| `fix` | 修复 Bug | `fix: 修复模型加载失败问题` |
| `docs` | 文档更新 | `docs: 更新部署说明` |
| `style` | 代码格式（不影响功能） | `style: 统一缩进格式` |
| `refactor` | 重构（不是新功能也不是修复） | `refactor: 优化预测逻辑` |
| `perf` | 性能优化 | `perf: 减少内存占用` |
| `test` | 添加测试 | `test: 添加单元测试` |
| `chore` | 构建/工具变更 | `chore: 更新依赖版本` |

### 示例

```powershell
# 简单提交
git commit -m "feat: 添加图片预处理增强功能"

# 带详细说明
git commit -m "fix: 修复 Streamlit Cloud 模型下载失败

- 添加 huggingface_hub 依赖到 requirements.txt
- 增加下载重试机制（最多 3 次）
- 改进错误提示信息"
```

## 三、完整命令参考

### 3.1 查看状态

```powershell
# 查看当前状态
git status

# 查看具体更改内容
git diff

# 查看某个文件的更改
git diff app.py

# 查看提交历史
git log --oneline -10
```

### 3.2 暂存文件

```powershell
# 暂存所有更改
git add .

# 暂存特定文件
git add app.py model_service.py

# 暂存某类文件（如所有 Python 文件）
git add *.py

# 交互式暂存（选择部分更改）
git add -p
```

### 3.3 提交

```powershell
# 提交暂存的更改
git commit -m "提交信息"

# 跳过暂存，直接提交所有已跟踪文件的更改
git commit -am "提交信息"

# 修改上一次提交（慎用，仅限本地未推送的提交）
git commit --amend -m "新的提交信息"
```

### 3.4 推送

```powershell
# 推送到远程主分支
git push origin main

# 强制推送（仅限特殊情况，如 rebase 后）
git push origin main --force-with-lease
```

### 3.5 撤销操作

```powershell
# 撤销暂存（保留更改）
git restore --staged <file>

# 撤销所有暂存
git restore --staged .

# 放弃工作区更改（危险操作！）
git restore <file>

# 撤销最近一次提交（保留更改）
git reset --soft HEAD~1

# 撤销最近一次提交（丢弃更改，危险！）
git reset --hard HEAD~1
```

## 四、Streamlit Cloud 部署

### 4.1 自动部署

推送到 GitHub 后，Streamlit Cloud 会自动触发重新部署：

1. 访问 https://share.streamlit.io/
2. 查看应用状态（通常显示 "Running" 或 "Deploying"）
3. 点击应用名称查看部署日志

### 4.2 手动触发部署

如果自动部署未触发：

1. 登录 Streamlit Cloud
2. 进入应用设置
3. 点击 "Reboot" 或 "Redeploy"

### 4.3 查看部署日志

部署失败时，查看日志定位问题：

1. Streamlit Cloud → 应用详情
2. 点击 "Logs" 标签
3. 查看错误信息

常见错误：
- `ModuleNotFoundError` → 检查 `requirements.txt` 是否包含所需依赖
- `FileNotFoundError` → 检查文件路径和模型下载逻辑
- `MemoryError` → 模型或数据太大，需要优化

## 五、依赖管理

### 5.1 添加新依赖

当使用了新的 Python 包时：

1. 更新 `requirements.txt`：
   ```
   # 在文件末尾添加
   新包名>=版本号
   ```

2. 提交更改：
   ```powershell
   git add requirements.txt
   git commit -m "chore: 添加 xxx 依赖"
   git push origin main
   ```

## 六、常用场景示例

### 场景 1：修改 Web 界面

```powershell
# 1. 修改 app.py

# 2. 本地测试
streamlit run app.py

# 3. 确认无误后提交
git add app.py
git commit -m "feat: 优化界面布局和配色"
git push origin main
```

```powershell
# 1. 修改 model_service.py 或其他相关文件

# 2. 提交
git add model_service.py
git commit -m "refactor: 优化预测后处理逻辑"
git push origin main
```

### 场景 3：更新文档

```powershell
git add README.md 技术实现.md
git commit -m "docs: 更新项目文档"
git push origin main
```


## 七、Git 代理设置（国内网络）

如果推送速度慢或失败，检查代理设置：

```powershell
# 查看当前代理
git config --global http.proxy

# 设置代理（Clash 端口 7897）
git config --global http.proxy http://127.0.0.1:7897
git config --global https.proxy http://127.0.0.1:7897

# 取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```

## 八、常见问题

### Q1: 推送被拒绝（rejected）

```powershell
# 远程有新提交，需要先拉取
git pull origin main --rebase
git push origin main
```

### Q2: 忘记添加某个文件

```powershell
# 追加到上一次提交（未推送时）
git add <file>
git commit --amend --no-edit
git push origin main
```

### Q3: 提交信息写错了

```powershell
# 修改上一次提交信息（未推送时）
git commit --amend -m "新的提交信息"
git push origin main
```

### Q4: Streamlit Cloud 显示旧版本

1. 清除 Streamlit Cloud 缓存：Settings → Clear cache
2. 手动触发重新部署
3. 检查 GitHub 仓库是否是最新的

### Q5: 需要回退到之前版本

```powershell
# 查看提交历史
git log --oneline

# 回退到指定提交（保留更改）
git reset --soft <commit-hash>

# 回退到指定提交（丢弃更改，危险！）
git reset --hard <commit-hash>

# 强制推送（如果已推送过）
git push origin main --force
```
